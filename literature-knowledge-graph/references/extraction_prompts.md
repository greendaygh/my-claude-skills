# LLM 추출 프롬프트 템플릿

## 추출 시스템 프롬프트

Claude가 전문에서 엔티티와 관계를 추출할 때 사용하는 프롬프트 구조.

## 1. 메인 추출 프롬프트 (스키마 기반 동적 생성)

```
당신은 과학 문헌에서 구조화된 지식을 추출하는 전문가입니다.

## 스키마 정의

다음 스키마에 따라 엔티티와 관계를 추출하세요:

### 엔티티 타입:
{schema.entity_types를 반복하여 생성}
- **{label}**: {description}
  - Primary key: {primary_key}
  - 속성: {properties}

### 관계 타입:
{schema.relationship_types를 반복하여 생성}
- **{type}** ({from_label} → {to_label}): {description}
  - 속성: {properties}

## 추출 규칙

1. **엔티티 이름 정규화**: 공식 명칭/기호 사용 (예: "p53" → "TP53", "breast cancer" → "Breast Cancer")
2. **신뢰도 기준**:
   - high (≥0.8): 실험적으로 확인된 직접 증거
   - medium (0.5-0.8): 추론적/간접 증거
   - low (<0.5): 가설/추측
3. **출처 섹션 태깅**: 각 추출 항목에 출처 섹션 기록
4. **스키마 외 발견**: 스키마에 없는 엔티티/관계도 "_unschemaed" 태그로 별도 기록

## 논문 전문

### {section_name}:
{section_text}

## 출력 형식 (JSON)

```json
{
  "entities": [
    {
      "label": "엔티티타입",
      "properties": {
        "name": "정규화된이름",
        "추가속성": "값"
      }
    }
  ],
  "relationships": [
    {
      "type": "관계타입",
      "from": {"label": "타입", "key": "이름"},
      "to": {"label": "타입", "key": "이름"},
      "properties": {
        "confidence": 0.9,
        "source_section": "results",

      }
    }
  ],
  "unschemaed": [
    {
      "description": "스키마에 없는 발견 설명",
      "suggested_entity_type": "제안 라벨",
      "suggested_relationship_type": "제안 관계 타입",
      "evidence": "근거"
    }
  ]
}
```

## 2. 섹션별 추출 지침

### Abstract
```
이 논문의 초록에서 핵심 발견을 추출하세요.
초록은 연구의 전반적 요약이므로:
- 주요 연구 대상 엔티티 식별
- 핵심 결론에 해당하는 관계 추출
- 신뢰도: 초록의 결론은 일반적으로 high
- 주의: 원 출처가 아닌 배경 설명은 구분 표기
```

### Introduction
```
서론에서 배경 지식 관계를 추출하세요.
서론의 특성:
- 이 논문의 원래 발견이 아닌 기존 지식 참조
- 신뢰도: high (확립된 지식) 단, is_background: true 표기
- 원 출처 논문이 있으면 참조 기록
```

### Methods
```
방법 섹션에서 실험적 맥락을 추출하세요:
- 사용된 세포주, 모델 생물, 약물 처리 조건
- 실험 기법/도구
- 이것은 "사실" 관계가 아닌 "실험 조건" 관계
- 관계 타입 예: TESTED_IN, TREATED_WITH, MEASURED_BY
```

### Results
```
결과 섹션은 가장 중요한 추출 대상입니다:
- 실험적으로 확인된 관계 → confidence: high
- 정량적 데이터 포함 시 속성에 기록 (p-value, fold change 등)
- 그림/표 참조 → figures_tables 기록
- 직접 실험 결과 vs. 통계적 추론 구분
```

### Discussion
```
토론 섹션에서 해석적 관계를 추출하세요:
- 추론적/해석적 관계 → confidence: medium
- "suggest", "may", "likely" 등의 헤징 표현 주의
- 다른 연구와의 비교/대조 관계
- 한계점 언급 → 신뢰도 하향 조정 근거
```

### Figures/Tables
```
그림과 표 캡션/범례에서 추출하세요:
- 표의 정량적 데이터는 높은 가치
- 그림 캡션의 핵심 결과
- 보충 자료 참조 여부 기록
```

## 3. 사이클 2 보완 추출 프롬프트

```
## 보완 추출 (사이클 2)

이전 추출 결과를 참고하여 **추가 엔티티/관계**를 추출하세요.

### 이전 추출에서 발견된 항목 (중복 방지):
{기존 추출 결과 요약}

### 새로 추가된 스키마 항목:
{schema_delta에서 추가된 엔티티/관계 타입}

### 임무:
1. 새로 추가된 엔티티/관계 타입에 해당하는 항목 추출
2. 기존 추출에서 누락된 항목 보완
3. 기존 항목과의 중복은 제외

이전 추출 결과와 **다른** 새로운 항목만 출력하세요.
```

## 4. 추출 품질 기준

| 기준 | 설명 |
|------|------|
| 정확성 | 원문에 명시된 관계만 추출 (과잉 추론 금지) |
| 완전성 | 스키마에 해당하는 모든 관계 포착 |
| 정규화 | 엔티티 이름 표준화 (동의어 통일) |
| 출처 추적 | 모든 항목에 source_section + evidence |
| 신뢰도 보정 | 섹션별 기본 신뢰도 + 헤징 표현 감점 |

## 5. 신뢰도 보정 규칙

```
기본 신뢰도 (섹션별):
  Results (직접 실험): 0.9
  Results (통계적 추론): 0.8
  Abstract (결론): 0.85
  Introduction (배경): 0.8 (is_background: true)
  Discussion (해석): 0.6
  Discussion (가설): 0.4

보정 요소:
  전문 사용: +0 (기준)
  초록만 사용: -0.2
  헤징 표현 ("may", "suggest"): -0.1
  정량적 데이터 동반: +0.05
  여러 실험에서 확인: +0.05
  단일 실험: +0
  모순 증거 있음: -0.15
```
