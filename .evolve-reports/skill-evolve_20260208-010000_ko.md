### 상세 개선사항

#### 1. 컨텍스트 윈도우 / 토큰 고갈 (치명적, 20/20 만장일치)
- **범주:** 프로세스
- **현재:** 최대 14명 고정 패널, 토큰 예산 안내 없음. 분석 시작 전 1506줄의 스킬 정의가 소비됨.
- **제안:** 3단계 동적 패널 크기 조정:
  - **단순** (스킬 < 100줄): Core 4명 + Domain 0명, 토론 1라운드
  - **표준** (100-300줄): Core 4명 + Domain 2-4명, 2라운드
  - **복잡** (300줄 이상): Core 4명 + Domain 4-16명, 전체 3라운드
  - 명시적 단어 예산 추가: Round 1에서 전문가당 150단어, Round 2에서 클러스터당 300단어
- **근거:** 이것 없이는 실제 사용이 불가능. 토큰 이코노미, 효율성, 프로세스 디자이너, UX 리서처 모두 독립적으로 식별.
- **위치:** SKILL.md 패널 구성 섹션 + Phase 2

#### 2. 입력 검증 / 경로 인젝션 (치명적, 20/20 만장일치)
- **범주:** 안전성
- **현재:** `$SKILL_NAME` 검증 없음. `../../.ssh/config`나 `name; rm -rf /` 같은 값으로 경로 탐색이나 명령어 인젝션 가능.
- **제안:** 인수 파싱 후 검증 블록 추가:
  1. `SKILL_NAME`이 `^[a-zA-Z0-9_-]+$`와 일치하는지 확인
  2. `~/.claude/skills/$SKILL_NAME/` 디렉토리 존재 확인
  3. 그 안에 `SKILL.md` 존재 확인
  4. 실패 시: 사용 가능한 스킬 목록과 함께 명확한 에러 메시지
- **근거:** 보안 전문가, QA 엔지니어, 사용성, API 설계 전문가 모두 독립적으로 지적.
- **위치:** SKILL.md, "Parse $ARGUMENTS" 섹션 다음

#### 3. 수렴 임계값 (치명적, 19/20)
- **범주:** 프로세스
- **현재:** 100% 수렴 필요 -- 모든 전문가가 "더 이상 개선 불필요"에 동의해야 함.
- **제안:** 85% 만족 임계값. 나머지 우려사항은 문서화하되 수렴을 차단하지 않음. Core 전문가만 차단 권한 보유 (Core 3/4 반대 = 조건부).
- **근거:** 14명 이상 패널에서 100%는 비현실적. 합의 도출 및 퍼실리테이션 전문가 모두 초다수결 옹호.
- **위치:** SKILL.md Phase 4, 479행

#### 4. 자기 참조 진화 (치명적, 19/20)
- **범주:** 안전성
- **현재:** `/skill-evolve skill-evolve` 호출 시 특별 처리 없음. 실행 중 자기 자신을 읽고 수정 가능.
- **제안:** `SKILL_NAME == "skill-evolve"`일 때:
  1. 분석 전 현재 스킬의 동결 복사본 생성
  2. 라이브 버전이 아닌 동결 복사본 분석
  3. 모든 라운드 완료 후에만 변경 적용
  4. 자동 수렴 루프 건너뛰기 (사용자가 직접 diff 검토)
  5. 사용자 승인을 위한 나란히 보기 diff 제시
- **근거:** 메타스킬 전문가가 논리적 역설 식별; QA 엔지니어가 정의되지 않은 동작 지적.
- **위치:** SKILL.md Phase 1, 새 섹션
